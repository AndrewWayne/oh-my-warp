# v0.4-thin BYORC Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Land a runnable BYORC slice — `omw-remote` daemon serving a Web Controller PWA over Tailscale Serve, paired via QR, attaching to a real shell PTY — without the agent integration (which depends on v0.2 + v0.3).

**Architecture:** Adopt **Option A** from the planning conversation: `vendor/warp-stripped/` is the canonical Warp host (tracked snapshot, not submodule). `omw-server` lives at outer `crates/omw-server/` (MIT) and is consumed by `vendor/warp-stripped/app/` via path dep — embedded omw-server. `omw-remote` runs as a sibling Rust binary, subscribes to omw-server's internal session registry over loopback, and bridges PTY frames to the Web Controller. v0.4-thin spawns a direct shell PTY (no Warp-pane PTY adapter) so the slice is independent of v0.2/v0.3 deliverables.

**Tech Stack:** Rust (axum, tokio, tower-http, ed25519-dalek, blake3, portable-pty, jsonwebtoken-or-paseto, rusqlite), TypeScript (Vite + React + TS + Tailwind, xterm.js, idb, zxing-js for QR scan), Tailscale Serve for tailnet exposure.
